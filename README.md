# 🎧 Опрос:  Использование подкастов 💖

Мой проект на Python — интерактивное веб-приложение для учебной практики, которое собирает ответы респондентов об использовании подкастов в повседневной жизни и сразу же строит наглядную аналитику для исследователя.

Всё работает на фреймворке **Streamlit**, собранные данные в реальном времени сохраняются в облачную NoSQL базу **Google Firebase (Firestore)**, а графики генерируются через **Plotly Express**. Интерфейс приложения оформлен в уникальном кастомном розовом стиле с градиентными шапками и кнопками, чтобы анкета выглядела эстетично и привлекательно для пользователей.

---

## 🛠️ Что умеет это приложение

1. **Принимать ответы:** Интерактивная и удобная анкета на 12 вопросов (возраст, пол, род занятий/социальный статус, платформы, любимые темы, оценка образовательного эффекта и др.).
2. **Разделять интерфейс:** Благодаря системе вкладок, анкета для обычных пользователей и панель аналитики для преподавателя находятся в одном приложении, но не мешают друг другу.
3. **Считать статистику:** На дашборде администратора мгновенно обновляются KPI-метрики (общее количество респондентов, средний балл полезности подкастов, средний возраст аудитории).
4. **Показывать графики:** Визуализирует данные с помощью 5 умных графиков:
   * Распределение оценок полезности (гистограмма).
   * Восприятие подкастов как инструмента обучения (круговая диаграмма).
   * Топ-7 самых популярных стриминговых платформ (горизонтальный бар-чарт).
   * Топ-7 востребованных тематик (горизонтальный бар-чарт).
   * **Сложная аналитика:** Диаграмма «Ящик с усами» (Boxplot), которая наглядно показывает связь между социальным статусом (студент, работающий, школьник) и тем, насколько высоко человек оценивает пользу подкастов.
5. **Выгружать данные:** Кнопка для скачивания всей сырой базы ответов напрямую из Firebase в один клик в формате CSV (совместимо с Excel).

---

## 📂 Как устроен проект (файлы)

* `app.py` — весь исходный код приложения (стили, форма анкетирования, подключение к БД и логика дашборда).
* `requirements.txt` — список библиотек, необходимых для корректной работы программы.
* `serviceAccountKey.json` — секретный ключ авторизации для подключения к Firebase (его на GitHub не заливаем!).
* `.gitignore` — файл, который приказывает Git игнорировать секретный ключ, чтобы защитить базу данных от утечки.

## ☁️ Как задеплоить в Streamlit Cloud

Когда будешь загружать проект на GitHub, заливай только app.py, requirements.txt, .gitignore и README.md. Файл serviceAccountKey.json на GitHub кидать нельзя.

Вместо этого настроим секреты в облаке:
1. Зайди в настройки своего запущенного приложения на сайте Streamlit Cloud.
2. Открой вкладку **Secrets**.
3. Скопируй туда данные из своего файла `serviceAccountKey.json`, переведя их в формат TOML вот так (просто заполни своими значениями вместо многоточий):

```toml
[FIREBASE_KEY] = '''
{type = "service_account"
project_id = "твой-id-проекта"
private_key_id = "..."
private_key = "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
client_email = "..."
client_id = "..."
auth_uri = "[https://accounts.google.com/o/oauth2/auth](https://accounts.google.com/o/oauth2/auth)"
token_uri = "[https://oauth2.googleapis.com/token](https://oauth2.googleapis.com/token)"
auth_provider_x509_cert_url = "[https://www.googleapis.com/oauth2/v1/certs](https://www.googleapis.com/oauth2/v1/certs)"
client_x509_cert_url = "..."
universe_domain = "googleapis.com"}'''
```

4. Нажми **Save**, и сайт в облаке заработает.

## 🔧 Что делать, если база данных выдает ошибки:

* **Если пишет Ошибку 403 (Cloud Firestore API... disabled):** Зайди в консоль Firebase, нажми слева на **Build -> Firestore Database** и нажми кнопку **Create database**. На этапе правил выбери **Start in test mode** (Тестовый режим).
* **Если пишет Ошибку 404 (database (default) does not exist):** Перейди по ссылке из текста ошибки в консоль Google Cloud. Выбери режим **Cloud Firestore в Native Mode** и подтверди создание базы данных по умолчанию с ID `(default)`. Подождите 2 минуты, пока сервера настроятся.
